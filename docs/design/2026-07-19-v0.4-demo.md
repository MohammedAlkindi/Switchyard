## Context

Switchyard's demo narrative changes exactly once between now and v1: when claims
land. Until then the story is "`fleet check` catches the overlap before you
merge." After claims it becomes "the overlap surfaced before either agent wrote
a line" — earlier, and a different pitch. This document designs the second one.

It does **not** cover the interim demo. v0.3 ships `fleet mcp` plus a Claude Code
skill, neither of which a terminal recording can show: a stdio JSON-RPC server
either idles silently or emits raw protocol frames. So v0.3 adds nothing
demoable, and the v0.2 `fleet check` flow is re-recorded now, separately, against
the current tape.

This design was originally written against
[design/2026-07-19-v0.3-fleet-protocol.md](2026-07-19-v0.3-fleet-protocol.md),
which places claims in v0.3. The plan supersedes that on scope.

**Resolved 2026-07-19:** the plan won. 0.3.0 shipped four read-only tools and no
claims; the protocol design is now marked partially superseded on scope. This
document's schedule stands as written — it remains blocked on claims, now
genuinely v0.4 rather than contingent on an open disagreement.

## Goals

- Land one idea: an overlap surfaced *before* either agent wrote a line.
- Demo only shipped behavior, rendered from a build of this checkout.

## Non-goals

- Showing `fleet mcp`. Documented in the README with a config snippet and the
  tool table instead.
- Showing `fleet check`. Its merge-time gate still matters, but a second payoff
  dilutes the arc and pushes past the runtime budget.
- Showing `fleet init`. Onboarding belongs in the quickstart, where readers
  arrive already intending to install.

## Narrative

Agent names stay `claude` and `codex`: the README's origin story is those two
colliding on one checkout, so the names carry meaning that `alpha`/`beta` would
not.

| Beat | Command | Proves |
| --- | --- | --- |
| 1 | `fleet spawn claude` / `fleet spawn codex` | the isolation premise — why there are two agents at all |
| 2 | `fleet claim claude src/api/ --note "routes layer"` | intent is declarable |
| 3 | `fleet claim codex src/api/routes.ts` | **overlap, at claim time** |
| 4 | `fleet claim codex src/lib/billing.ts --replace` | routing around — the tool changes a decision |
| 5 | `fleet claims` | resulting state: two agents, disjoint territory |

Closing card: `# overlap surfaced before either agent wrote a line`

Beat 1 is retained from the existing tape because claims are abstract
bookkeeping to a viewer who has not yet seen that each agent gets its own
worktree. Beat 4 uses `--replace` so the route-around is one command rather than
an `unclaim` / `claim` pair.

The scenario exercises the path-component-prefix case (`src/api/` overlapping
`src/api/routes.ts`) that the protocol design singles out as the one naive
string matching gets wrong — not a trivial identical-path match.

## Dependency — warn at claim time

This design assumes **`fleet claim` reports an overlap when the claim is
recorded.** The protocol design specifies overlap annotations on `fleet claims`
only; it does not say `claim` itself warns.

If `claim` stays silent, beat 3 cannot land: the demo needs an intervening
`fleet claims` call to reveal the overlap, and "codex saw it immediately"
degrades to "codex ran a second command to find out."

**Recommended amendment to the claims feature, for decision:** have `fleet claim`
print overlap warnings on write. The rationale is not the demo. An agent that
must run a second command to discover it is trespassing will often not run it,
and the information is only actionable at the moment of declaration. This is
additive, and does not change the advisory model, exit codes, or the claims file
schema.

## Production

Unchanged from the v0.2 tape: Tokyo Night, 1920x1080 at 28pt JetBrains Mono,
`Hide`/`Show` blocks for repo setup, container-only rendering because the setup
runs `git config --global`.

```sh
docker build -f Dockerfile.vhs -t vhs-fleet .
docker run --rm -v "$PWD:/vhs" vhs-fleet demo.tape
ffmpeg -i assets/demo.mp4 -vf "fps=10,scale=960:-1:flags=lanczos,split[s0][s1];[s0]palettegen=max_colors=128[p];[s1][p]paletteuse=dither=bayer:bayer_scale=5" -loop 0 assets/demo.gif
```

Runtime lands at ~35-40s, above the v0.2 demo's 30s: six visible commands at
55ms/char plus dwell time to read the overlap warning. Trimmable to ~32s by
cutting the spawn dwells to 2s and dropping beat 5. Prefer 35s over a rushed 30 —
beat 3 is the payoff and needs time to land.
